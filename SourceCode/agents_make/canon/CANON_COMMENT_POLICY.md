# Canon Comment Policy

Apply these rules in every slot fill.

- Python file-level docstring: exactly one line describing file purpose.
- Python function docstring: one line for obvious behavior; 2-4 lines only when needed.
- Do not add `Args:` or `Returns:` sections when typed signatures are self-evident.
- Inline comments are allowed only for non-obvious logic.
- Do not write comments that restate the code.
- JavaScript: JSDoc only for exported helpers; keep inline comments sparse.
- SQL comments should annotate non-obvious constraints or indexes only.
- Never write author-attribution phrases such as "AI-generated", "generated by", or similar.
